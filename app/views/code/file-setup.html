File setup

For this tutorial, we will be recreating an already live component known as Currency Input. This component gives context to the user that the number they will be typing in is a monetary amount, ensuring they are aware in the case that they will be paying for something online. This is how Currency Input looks:

To start this tutorial, we will first need to set up our work environment, creating some folders and files that we can work in.


  • Open your terminal, and navigate to the hmrc-frontend GitHub repository that we cloned earlier. You can do this with the cd terminal command.
  • Next, create a branch of hmrc-frontend. This is a version of the repository that you can make changes to and it will not affect the main version. You should give this a suitable name, such as 'yourname-placement-tutorial'. The following command may help:
    • {{ codeSnippet({ code: 'git checkout -b yourname-placement-tutorial' }) }}
  • This will create the branch with that name, and automatically move you on to it. Now we can type the following command to open our workspace.
    • {{ codeSnippet({ code: 'code .' }) }}
  • Navigate to src/components. You will see there is a folder for each component inside hmrc-frontend. You should create your own with a name like 'tutorial-component'.
  • We can make the files in this folder that will make up our component. For each of the following file names, right click your new folder and create a file:
    • template.test.js
    • tutorial-component.yaml
    • template.njk
    • macro.njk
    • _tutorial-component.scss

This is all the set up we will need. In the next section, we can move on to creating our component, beginning with writing some tests to cover all aspects of Currency Input.
