File setup

For this tutorial, we will be recreating an already live component known as Timeline. This component is a handy visual that can provide the user with a clearly defined timeline, which could be used to show different dates of payment, or dates in which a HMRC process will be completed in stages. Most of the code is provided for you, and you will be tasked with trying to write a section yourself, provided with some hints and reference to the real component code. The styling has also been fully provided. This is how Timeline looks:



To start this tutorial, we will first need to set up our work environment, creating some folders and files that we can work in.


  • Open your terminal, and navigate to the hmrc-frontend GitHub repository that we cloned earlier. You can do this with the cd terminal command, followed by the filepath.
  • Next, create a branch of hmrc-frontend. This is a version of the repository that you can make changes to and it will not affect the main version. You should give this a suitable name, such as 'yourname-placement-tutorial'. The following command may help:
    • {{ codeSnippet({ code: 'git checkout -b yourname-placement-tutorial' }) }}
  • This will create the branch with that name, and automatically move you on to it. Now we can type the following command to open our workspace.
    • {{ codeSnippet({ code: 'code .' }) }}
  • Navigate to src/components. You will see there is a folder for each component inside hmrc-frontend. You should create your own with a name like 'tutorial-component'.
  • We can make the files in this folder that will make up our component. For each of the following file names, right click your new folder and create a file:
    • template.test.js
    • tutorial-component.yaml
    • template.njk
    • macro.njk
    • _tutorial-component.scss
  • Finally, add the changes made to the branch. This can be done with the following commands:
    • {{ codeSnippet({ code: 'git add --all git commit -m "Initial file set up" git push' }) }}

This is all the set up we will need. In the next section, we can move on to creating our component, beginning with writing some tests to cover all aspects of Timeline.
