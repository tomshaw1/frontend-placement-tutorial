Writing YAML

Another important attribute of a HMRC component is the YAML configurations. We use YAML to set parameters for a component; this means that we can make them customisable, rather than being statically written up, we give service teams more freedom to make components fit for their specific service.

Below is the first of two required YAML items, 'params'. You should add this to your tutorial-component.yaml file.


To provide some context to this code, this is the params item. This has two children, events (an array) and headingLevel (an integer). In terms of the structure of the code, YAML uses white space indentation to distinguish parent and child nesting, and the '-' is used for an item.

The 'events' has its own 'params' item. This is because each event has a multitude of required items, like a title, the date, and description of the event. In the application of this component, the developer will fill the events array with objects containing all the relevant information for each event, and the timeline will populate using this data.

The 'headingLevel' is an integer used to provide more customisation to the timeline. The user can set this to customise what heading type is used, for example h1, h2, h3.

Writing the examples

Your job is to try and write the second and final main item required for our yaml file, 'examples'. You may recognise this from the tests; we will make items such as 'default', 'single-event' for use in testing. Following the syntax and structure of the above item, you can put this together with the following information:

Required items:

  • default
  • custom-header-level
  • no-datetime-event
  • single-event
  • mulitple-events

Required key-value pairs in each item:

  • title - for this attribute, write an appropriate title (not a string)
  • description - for this attribute, write an appropriate description of the example (not a string)
  • data - this will be an array to contain the required info for an event. Each event will be defined by it's title, which is a string
  • each event will require a time, which is a string containing a human readable date (i.e '15 July 2020')
  • each event will require content, which is a string containing html
    (i.e '<p class="govuk-body">singleExampleContent1</p>')

Try and write up the remainder of this code. If you are having trouble, you can see the real Timeline YAML file for help. Once you have completed it, running npm start in your terminal will open hmrc-frontend in your browser. Scrolling down to your Tutorial Component, you should see corresponding links for each of the examples you made!
