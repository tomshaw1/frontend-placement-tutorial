YAML

What is YAML?

YAML is a data-serialisation language, used in HMRC to create and test several configurations of a component dynamically, so that we do not have to change the content of the actual code.

For example, we can have a list pattern, and use YAML to create an example with 1 item in the list, and with 20 items in the list.

Another key use of YAML is to create parameters for a component. This means that we can make the component more of a template, and the parameters we provide will allow service teams to customise certain aspects of the component.

'single-event' creates an example with a single event, and all the data that is needed to populate said event, such as the title and date. 'multiple-event' is very similar, instead containing two events.

These different configurations can be used to visually check a component is showing different types of data properly, and can be used to create unit tests. This is an important step in the process of creating a component and is often done alongside the coding of the component itself.
